# 自社Webサイト構築プロジェクト

## プロジェクト概要

BtoBのITサービス・SaaS企業として、2〜4つのサービスを訴求するコーポレートサイトを新規構築するプロジェクトです。

## 要件まとめ

| 項目 | 内容 |
|------|------|
| 業種 | ITサービス・SaaS |
| ターゲット | BtoB（企業・法人） |
| サービス数 | 2〜4つ |
| デザイン | プロフェッショナル・信頼感重視 |
| コンテンツ更新 | 非エンジニアも更新したい |
| ブランドアセット | ロゴあり（カラー・コピーはこれから） |
| 必須コンテンツ | 会社概要・ミッション、サービス紹介、導入事例・実績、お問い合わせフォーム |

## 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フロントエンド | Next.js (App Router) | SEO・パフォーマンス。静的生成とSSRのハイブリッド対応 |
| スタイリング | Tailwind CSS | 一貫したデザインシステムを素早く構築できる |
| CMS | microCMS | 日本製ヘッドレスCMS。非エンジニアでも直感的に操作可能 |
| ホスティング | Vercel | Next.jsと最高の相性。CDN・デプロイ自動化が無料枠で使える |
| フォーム | Resend + API Route | 問い合わせメール送信。シンプルかつ信頼性が高い |

## サイト構成

```
/ (トップページ)
  ├── Hero セクション（キャッチコピー + CTA）
  ├── サービス概要（カード形式で2〜4つ）
  ├── 導入事例ピックアップ
  └── 問い合わせCTA

/services (サービス一覧)
/services/[slug] (各サービス詳細)

/company (会社概要)

/cases (導入事例一覧)
/cases/[slug] (各事例詳細)

/contact (お問い合わせ)
```

## 実装フェーズ

| フェーズ | 内容 |
|---------|------|
| Phase 1 | プロジェクト初期設定（Next.js + Tailwind + microCMS連携） |
| Phase 2 | 共通コンポーネント（Header, Footer, レイアウト） |
| Phase 3 | 各ページ実装（トップ → サービス → 会社概要 → 事例 → 問い合わせ） |
| Phase 4 | CMS接続・コンテンツ入力 |
| Phase 5 | Vercelデプロイ・ドメイン設定・最終確認 |
